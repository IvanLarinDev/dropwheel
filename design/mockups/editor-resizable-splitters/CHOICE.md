# Редактор цели — изменяемый размер и сплиттеры

- Дата: 2026-07-15
- Одобрил: Ivan Larin
- Выбран: вариант C — сплиттеры между тремя колонками плюс горизонтальный сплиттер над превью, в изменяемом по размеру окне.

## Что менялось

Окно редактора цели в виде сортировщика (когда открыт редактор правил) стало изменяемым по
размеру. Три колонки — поля цели, список правил, деталь правила — переведены на звёздные
ширины с двумя вертикальными `GridSplitter` между ними; над панелью превью добавлен
горизонтальный `GridSplitter`, чтобы превью можно было растянуть под длинные списки
совпадений. Внутренние `ScrollViewer` больше не ограничены фиксированной высотой — они
заполняют свои панели и растут с окном.

Переключение в изменяемый режим делает код (`EnableResizableLayout` в момент показа
редактора правил): `SizeToContent=Manual`, `ResizeMode=CanResize`, стартовый и минимальный
размер, звёздные колонки/строка и показ сплиттеров.

## Простая цель не меняется

Для простой цели (папка или приложение без правил) редактор остаётся компактным и
фиксированным по контенту, как раньше — пустые колонки правил/детали и строка превью в этом
режиме не резервируют места. Изменяемый режим включается только для сортировщика.

## Сопутствующее

Каркас `DialogShell`: строка контента переведена с `Auto` на звёздную, чтобы окно тянулось
по высоте; для остальных окон это нейтрально (они `SizeToContent`). В эту же работу свёрнута
правка прокрутки списка правил к выбранной строке (`BringIntoView`).
