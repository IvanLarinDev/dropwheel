DialogShell — four ways to unify the dialogs

Every dialog (New group, Target editor, Delete group, Settings, and a new in-app message box) shares one frame. The button contract is fixed in all four: primary action on the right, Cancel to its left, an optional destructive action far left. What differs below is how the shell frames that footer and signals severity. Colors are the app's real Fluent theme.

Footer law (same in every variant): right edge holds the Primary verb (Save / Create / Delete group, keep items). Cancel sits directly left of it, bound to Esc; Primary is bound to Enter. A destructive action lives far left as a quiet danger-text button. Primary is disabled while the form is invalid.
1
Flat footer

Closest to today. No divider, no header — content sits straight above a right-aligned button row. Lightest touch, least new chrome. Severity carried only by the danger-colored button.

Divider + in-content header

A hairline separates content from the footer, and each dialog opens with a bold title line plus one-line subtitle — so a message box or a delete prompt is self-describing, not just a floating sentence. More structure, still calm.

3
Severity strip

A thin colored strip across the top reads the dialog’s role at a glance — accent blue for ordinary edits, danger red for destructive or error dialogs. The footer stays flat. Best when the message box needs to telegraph “this is a problem” instantly.
