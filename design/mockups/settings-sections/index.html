Settings — four ways to section it, one validation language

Today the 13 settings sit in one tall scrolling column. All four variants group them into the same four sections — Wheel behavior, Appearance, Hotkey & shortcuts, System — inside a fixed-height window. What differs is how a section is drawn. Validation is the same in every variant: errors appear inline under the field, Save is disabled while anything is invalid, and the hotkey field checks itself live (shown in the Hotkey section).

Validation, everywhere: no more pop-up message boxes. A bad value shows a red line under its own field and turns the field border red; the hotkey field shows a live “Available” / “Already taken” line as you type (your current hotkey always counts as available). Save stays disabled until every field is valid, and clicking it scrolls to the first problem.
