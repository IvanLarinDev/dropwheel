# Создание группы через тайл Add

- Дата: 2026-07-15
- Одобрил: Ivan Larin
- Выбран: вариант A — переключатель «Target / Group» в шапке редактора при создании, оформленный как pill-сегмент (таблетка из двух половин, выбранная подсвечена акцентом).

## Что менялось

Создание группы переехало из контекстного меню орба в тайл Add. Клик по тайлу Add
открывает редактор цели в режиме создания с переключателем типа «Target / Group» вверху
формы. По умолчанию Target — форма как раньше. Выбор Group скрывает поля пути, действия,
переименования, конфликта, родительской группы, launch-опций и кнопку Convert, оставляя
только имя, эмодзи, цвет, код группового шортката и «Pin». При сохранении группа
регистрируется в корне колеса; пустой код шортката для новой группы получает автоматический
следующий свободный.

Переключатель показывается только при создании: у существующей цели или группы тип менять
нельзя (есть путь или дети), поэтому редактирование работает как раньше — но редактор
существующей группы теперь тоже прячет нерелевантные поля цели (раньше показывал их
заблокированными).

## Что убрано

Пункт «New group…» из контекстного меню орба и метод `CreateGroup` с отдельным
мини-промптом — создание группы теперь единообразно идёт через тот же редактор, что и цель.

## Сопутствующие правки

- Pill-стиль переключателя вынесен в общий словарь `DialogShell.xaml` (подключён app-wide),
  а не в `Window.Resources` окна — иначе собственный блок ресурсов окна ломал подмешивание
  тёмных Fluent-словарей `ThemeMode` (поля становились светлыми). Ресурсы `Themes.ApplyWindow`
  расширены ролями `DwAccent/DwAccentText/DwSelection/DwSurface` по тому же паттерну.
- По итогам ревью критиков: окно редактора при «Convert to routing rules» больше не уезжает
  за край экрана — после роста уже показанного окна оно перецентрируется и клэмпится в рабочую
  область (`EnableResizableLayout`/`KeepOnScreenAfterGrow`). Плюс мелкая чистка (`_closeR` против
  дублирования числа зоны, разведение `OnSave` на `SaveGroupFields`/`SaveTargetFields`).
