# design/mockups — пакеты дизайн-гейта

Каждая папка здесь — след одного изменения интерфейса: по ней видно, что именно
менялось во внешнем виде и на каком основании. Гейт дизайна (pre-push) требует,
чтобы у любой правки `src/Dropwheel/UI/**` был такой пакет.

## Что лежит в пакете

Папка называется по фиче (`hotkey-chips`, `tile-colour`, …) и содержит одно из двух:

- `mockups.html` + `CHOICE.md` — когда визуал менялся: файл с вариантами макетов
  (обычно четыре) и запись, какой вариант выбран, кем и когда;
- `WAIVER.json` — когда правка UI-файла чисто логическая и картинку не меняет:
  объяснение, почему макеты не нужны, с датой и именем одобрившего.

## Как добавлять

Новое изменение внешнего вида: сначала показать владельцу варианты макетов, после
выбора положить сюда `mockups.html` и `CHOICE.md`. Логическая правка без изменения
картинки — только `WAIVER.json` по образцу соседних. Старые пакеты не редактировать
и не удалять — это история решений.
