# Services — заметка для нейросети

Роль папки: вся бизнес-логика без интерфейса — файлы, запуск, хуки ввода, слежение за
папками, сортировка, иконки, метаданные ссылок. Окна не открывает (кроме служебных
оконных подсказок захвата, живущих в `../UI`).

## Ключевые файлы

- `TargetStore.cs` — единая точка чтения/записи `config.json`. Внутри: атомарное
  сохранение через temp-файл, дедуп (`DedupKey`, `SplitNewAndDuplicates`), санитайзер
  неизвестных enum-ов (`TrySanitizeInvalidEnums`/`SanitizeTargets`/`SanitizeRules` — при
  добавлении enum обновляй его), удаление с чисткой фавикона (`DeleteTarget`). Тесты
  подменяют папку через `DirOverride`.
- `LaunchService.cs` — запуск целей. Интерпретаторные ветки (`.ps1/.py/.jar`) собирают
  аргументы через `ArgumentList` (не строкой) — это защита от инъекции из крафтового
  `.lnk`. `ShortcutResolver.cs` отсеивает подозрительные пути.
- `MouseHook.cs`/`KeyboardHook.cs` — низкоуровневые WH_*_LL хуки. Вызовы подписчиков
  обёрнуты в try/catch (исключение в нативном callback = падение процесса).
- `HotkeyService.cs` — глобальный хоткей; `AddHook` только после успешной регистрации.
- `WatcherService.cs` — авто-сортировка. Блокировку `_gate` НЕ держать во время
  `SHFileOperation` (иначе UI-поток морозится на выходе/снятии папки).
- `IconService.cs` — кэш иконок; кэшируются только успешные, декод под размер тайла.
- `LinkMetadataService.cs` — заголовок и фавикон ссылки; дисковый кэш проверяется до сети.
- `WheelLayout.cs`, `SortService.cs`, `DropDispatch.cs` — чистая геометрия/сортировка/
  приоритеты дропа (хорошо покрыты тестами). `SortService` сортирует и папки: у правила
  есть `Scope` (файлы/папки/оба), дата-токены читают дату папки через `FileSystemInfo`, а от
  петли слежения защищает `AlreadyPlaced` — папку с именем в форме назначения (созданную самим
  сортировщиком) не трогаем. `WatcherService` перечисляет `GetFileSystemEntries`.

## Правила именно здесь

- Никакого WPF-визуала и открытия окон отсюда. Ошибки — через `ErrorLog`, не глотать.
- Логику, которую стоит протестировать, делай `internal static` и чистой; проект открыт
  тестам через InternalsVisibleTo.
- Перед правкой сигнатур ищи вызовы через `ck` — сервисы дергает весь `../UI`.

## Связи
