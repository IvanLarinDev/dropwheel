# UI — заметка для нейросети

Роль папки: весь WPF-интерфейс — окно-оверлей с орбом и колесом, настройки, редактор
цели/правил, подсказки, темы. Логика без интерфейса — в `../Services`, данные — в
`../Models`.

## Ключевые файлы

- `OverlayWindow.*.cs` — одно окно, разбитое на `partial class` по темам:
  `Layout` (размер/позиция окна, ручное перемещение орба в DIP), `Cloud` (построение и
  анимация плиток; `TileElementsByTarget` — общий безопасный поиск тайла), `Bubble`/
  `Bubble.Wire`/`Plus`/`Back` (плитки и их события), `Dnd`/`OrbDrop`/`PinDrop`
  (перетаскивание; приоритеты дропа считает `DropDispatch` из сервисов), `Proximity`/
  `Charge` (реакция на приближение; rendering-loop стартует только при заряде),
  `Capture` (Alt+Shift-захват: координаты приводятся к DIP, есть гард повторного входа и
  «поколение» probe), `GroupShortcuts`, `Undo`, `IdleFade`, `Sort`, `Hotkey`.
- `SettingsWindow.xaml(.cs)`, `TargetEditorWindow.xaml(.cs)` + `TargetEditorWindow.Rules.cs`,
  `PromptWindow.xaml(.cs)` — прочие окна.
- `Themes.cs`, `Palette.cs`, `MenuTheme.xaml` — палитры и оформление.

## Правила именно здесь

- Изменения внешнего вида, раскладки и анимаций — только после мокапов: срабатывает хук
  `design-gate` (жёсткий гейт в pre-push/CI). Чисто логическая правка UI-файла (без
  изменения картинки) допустима, но при пуше потребует waiver с пояснением.
- Новую тему главного окна выноси в отдельный `OverlayWindow.<Тема>.cs`, не раздувай
  существующие части.
- Плитки рисуются из фигур без текста — задавай `AutomationProperties.Name` для
  доступности.
- Координаты курсора из `GetCursorPos` — это device-пиксели; для позиций окон приводи к
  DIP (`DeviceToDip`/`TryCursorDip`), иначе сдвиг на масштабе ≠ 100%.
- Геометрия/тайминги/цвета зеркалятся в `docs/demo` — меняешь тут, помни про демо.
